Content Authoring Tool

Client: MCG Health

MCG helps healthcare organizations implement informed care strategies that proactively and efficiently move patients toward health. Care guidelines from MCG provide fast access to evidence-based medicine’s best practices and care plan tools across the continuum of treatment, providing clinical decision support and documentation which enables efficient transitions between care settings

Duration : 6 months

Process: User Research , User Interviews, User Journey, Prototypes, Usability Testing, Hand-off and communication to delivery team.

My Role: As a Lead UX designer I led cross functional engagement with Product, Development and Project Management teams to re-design CAT tool. I solely executed the complete design process.

Tools : Sketch, InVision, Figma, Miro board, Confluence (for documentation), Jira (PI Planning)

Project Description


MCG provides a client facing Content Authoring Tool (CAT) to enable the Nurses to customize patients clinical assessment content. CAT allows clients to make changes to the original MCG content to meet any regulatory or internal organizational requirements.
MCG received a lot of negative feedback from customers about the usability of the CAT tool. My task was to understand the feedback and re-design the tool to fix these usability issues.

Challenges


Some of the challenges in this project are:

  • No prior domain knowledge.
  • Understanding how the current CAT tool works.
  • Not knowing the user's workflow.
  • Identifying end-user likes and dislikes about the tool.

Research


During the research phase I explored the following:

  • Understanding the workflow of UM (Utilization Management) Nurses.
  • Identifying missing features.

Project Goals:

  • Gather information on CAT users
  • Gather user feedback on the CAT tool
  • To empower UM (Utilization Management) Nurses by providing the best user experience.
  • Improve the experience for current and prospective users while preserving its value as a work tool for the team

Pain points:

  • Unhappy customers leads to revenue loss
  • Workflow challenges
  • Learning curve is steep
  • Misaligned resource expectations
  • Tool is clunky
  • Users have to create all sections first and then stitch together. Bottom-up approach is unintuitive to users.

Objective

Provide a feature rich, user friendly web application

  • Rethink Workflows
    • From bottom-up to top-down
    • Interactive Preview
  • Improve UI
    • Address actionable changes from customer feedback
    • Update CAT 1.0 mockups to match & test out Style Guide
    • Design for additional requirements
  • Reduce time to create a guideline content down to 15 minutes from 1 day

Re-design



The design process included the following steps:

  • User flow.
  • Usability Testing with clickable prototype.
  • Iterative designs.

User Flow

I created a user flow to understand how the users navigate through if the workflow is changed to their preferred top-down approach. I refined the user flow by collaborating with product owner and internal stakeholders who were UM nurse’s before.

Benefits of CAT UI Design

  • Increased win & relicense rate
    • An intuitive design for CAT is estimated to increase Care Coordination win rate +5% and relicense rate by 1%
  • Decreased development cost
    • Allowed us to perform UAT/gather feedback from internal & external SME’s prior to development
    • Identify duplicative work that we were able to remove from scope
      • New Section, +New Question perform same function as in-line editing
    • Development team doesn’t need to create icons, typography, buttons
  • Facilitated communication with development team
    • Jira tasks tied to UI frames & acceptance criteria
    • UI designer active participant in PI planning & refinement session

Takeaways:

  • Design effectively for the specific domain
    • Get to know the audience
    • Ask for feedback early
  • Establish process and rules
    • Style Guide is a good start
    • Design team files organization
  • Future usability testing on CAT 2.0
